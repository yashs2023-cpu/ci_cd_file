✅ STEP 5: Create Jenkins Pipeline Job
👉 New Item → Pipeline → Name: CI-CD-Demo

Paste this pipeline:

pipeline {
    agent any

    environment {
        DOCKER_IMAGE = "yourusername/html-demo"
    }

    stages {

        stage('Clone Code') {
            steps {
                git branch: 'main',
                    url: 'your-github-repo-link'
            }
        }

        stage('Build Docker Image') {
            steps {
                bat 'docker build -t %DOCKER_IMAGE% .'
            }
        }

        stage('Push Image') {
            steps {
                withCredentials([usernamePassword(
                    credentialsId: 'dockerhub',
                    usernameVariable: 'USER',
                    passwordVariable: 'PASS'
                )]) {
                    bat 'docker login -u %USER% -p %PASS%'
                    bat 'docker push %DOCKER_IMAGE%'
                }
            }
        }

        stage('Deploy to Kubernetes') {
            steps {
                bat 'kubectl apply -f deployment.yaml'
            }
        }
    }
}

❌ ERROR
cannot perform an interactive login from a non-TTY device
🎯 WHY THIS HAPPENS
👉 Jenkins runs in non-interactive mode
👉 But your command:

docker login -u USER -p PASS
tries to login interactively

🚫 That’s not allowed inside Jenkins

✅ FIX (VERY IMPORTANT)
🔥 Replace your login command
❌ WRONG (current)
docker login -u %USER% -p %PASS%
✅ CORRECT (use password-stdin)
echo %PASS% | docker login -u %USER% --password-stdin
✅ UPDATE YOUR PIPELINE
Change Push Image stage to this:

stage('Push Image') {
    steps {
        withCredentials([usernamePassword(
            credentialsId: 'dockerhub',
            usernameVariable: 'USER',
            passwordVariable: 'PASS'
        )]) {
            bat """
                echo %PASS% | docker login -u %USER% --password-stdin
                docker push %DOCKER_IMAGE%
            """
        }
    }
}
⚡ WHY THIS WORKS
👉 --password-stdin avoids interactive login
👉 Jenkins can pass password safely
